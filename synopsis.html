Abstract

VULNRECON is an advanced, full-stack cybersecurity intelligence platform developed to automate and streamline the process of network reconnaissance, vulnerability scanning, and threat analysis. Traditional scanning tools like Nmap generate vast amounts of raw data, placing a heavy cognitive burden on security analysts to manually filter false positives, correlate CVEs, and determine remediation paths.

VULNRECON addresses these challenges by orchestrating deep infrastructure scans through a decoupled Python FastAPI microservice, returning real-time structured data via Server-Sent Events (SSE) to a modern, glassmorphic Next.js dashboard. The platform's most distinguishing feature is its proprietary, zero-API Rule-Based AI Engine developed entirely in JavaScript. By abandoning external LLM cloud APIs, VULNRECON guarantees absolute data privacy, zero recurring usage costs, and immunity to rate limiting during critical incident response scenarios.

01.

Introduction

1.1 Background & Overview

In the rapidly evolving landscape of cybersecurity, the attack surface of organizations is expanding exponentially. Cloud migrations, the proliferation of Internet of Things (IoT) devices, and remote work infrastructure have created unprecedented challenges. Network reconnaissance—the process of identifying active hosts, open ports, and running services—remains the fundamental first step in both offensive penetration testing and defensive posture management.

1.2 Motivation

The primary motivation behind VULNRECON is the cognitive overload experienced by Security Operations Center (SOC) analysts. A typical enterprise scan can reveal thousands of open ports and potential CVEs. Analysts spend disproportionate amounts of time triaging false positives rather than neutralizing threats.

1.3 Problem Statement

  • 01. Data Overload: Massive, unstructured datasets are difficult to parse visually and prioritize.
  • 02. The "API Tax": Cloud LLMs like GPT-4 are expensive, rate-limited, and pose privacy risks.
  • 03. Lack of Actionability: Raw scan results lack specific, context-aware remediation commands.

1.4 Proposed Solution & Objectives

Automation

Remote orchestration of Nmap via FastAPI microservice.

Intelligence

Local Rule-Based engine for Zero-API threat analysis.

Actionability

Step-by-step shell commands for instant patching.

Reconnaissance

OSINT integration with Shodan and AbuseIPDB.

Literature Review

Existing security tools like Nmap are the gold standard for port discovery but remain inherently "dumb." They identify what exists but provide no context. Enterprise solutions like Nessus or Qualys are monolithic, expensive, and often produce static PDF reports that are hundreds of pages long, obfuscating critical issues.

3.3 Data Flow Architecture

1. Initiation: User submits a target (IP/Domain) via the React dashboard.

2. Proxy & Execution: Node.js proxies the request to FastAPI, which spawns an Nmap subprocess with flags -sV -sC -O -A.

3. Real-time Streaming: Nmap stdout is captured and streamed back to the frontend line-by-line via SSE.

4. Final Enrichment: Upon completion, the result is parsed into JSON and enriched by the internal lib/ai-engine.

05.

The Rule-Based AI Engine

The core innovation of VULNRECON is its Zero-API approach. By implementing six specialized JavaScript modules, the platform avoids the latency and privacy risks associated with cloud LLMs.

5.2 Module 1: Severity Predictor

Uses a weighted heuristic matrix analyzing five distinct vectors: CVE Presence (40%), Service Exposure (20%), Version Age (15%), Protocol Security (15%), and Port Sensitivity (10%).

const vectors = {
  CVE_WEIGHT: 0.4,
  SERVICE_EXPOSURE: 0.2,
  PROTOCOL_RISK: 0.15
};
let score = calculate(vectors, portData);

5.5 Module 4: Attack Path Analysis

Utilizes graph-based algorithmic traversal to map multi-step attacks, categorizing ports into Entry Points, Lateral Movement Vectors, and Data Targets.

5.6 Module 5: Automated Remediation Engine

Lookup tables provide precise shell commands based on the detected service and Operating System.

Target: SSH (22)

Edit /etc/ssh/sshd_config: Disable root login, PermitRootLogin no.

Target: FTP (21)

Migrate to SFTP/SCP. Ensure anonymous access is disabled in vsftpd.conf.

Target: MySQL (3306)

Bind-address to 127.0.0.1. Implement strict ACLs for remote access.

07.

Security Hardening

7.3 Command Injection Prevention

The application passed a rigorous audit for CWE-77. A stringent regex pattern ensures only safe alphanumeric characters and network delimiters are permitted.

const safePattern = /^[a-zA-Z0-9.\-:\/_]+$/;
if (!safePattern.test(trimmed)) {
  throw new Error('Invalid Target');
}
7.1 Authentication

128-bit JWT keys stored in httpOnly, Secure cookies. Immunity to XSS-based token theft.

7.2 Rate Limiting

Sliding-window throttles: 5 req/min for scans, 15 req/min for OSINT tools. Prevents Nmap worker exhaustion.

7.4 CSRF Mitigation

SameSite=Strict cookie flags ensure cross-origin requests cannot hijack user sessions.
